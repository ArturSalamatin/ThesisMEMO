# Библиография — полные правила оформления записей

Эталонный `.bib` пользователя — в `references/examples/references.bib`; держи его перед глазами как образец.

Движок — `biber` + `biblatex` со стилем `gost-numeric`. Все настройки уже сидят в преамбуле (см. `references/preamble.tex`, секция «БИБЛИОГРАФИЯ»). Цитирование — `numeric-comp` (сворачивает диапазоны: `[1,3,5]`, `[2--4]`). Сортировка — `none`, нумерация по порядку первого упоминания в тексте.

## 1. Ключи записей

- Формат `фамилияГГГГ`, нижний регистр, без пробелов и точек: `samarskii1989`, `toselli2005`, `cai1999`, `schwarz1870`.
- Год — полный 4-значный.
- Для коллективных авторов или особых случаев — короткое описательное имя: `matlab2023`, `schwartz_github`.
- Несколько работ одного автора в одном году — суффикс `a`, `b`: `samarskii1989a`, `samarskii1989b`.

## 2. Типы записей

Используются три типа. Других не вводи без необходимости.

### `@book` — монографии, учебники

Обязательные поля: `author`, `title`, `publisher`, `address`, `year`, `pagetotal`, `langid`.
Опциональные: `edition`.

Пример:
```bibtex
@book{samarskii1989,
  author = {Самарский, А. А.},
  title = {Теория разностных схем},
  publisher = {Наука},
  address = {Москва},
  year = {1989},
  pagetotal = {616},
  langid = {russian},
}
```

С переизданием:
```bibtex
@book{bahvalov2003,
  author = {Бахвалов, Н. С. and Жидков, Н. П. and Кобельков, Г. М.},
  title = {Численные методы},
  publisher = {БИНОМ. Лаборатория знаний},
  address = {Москва},
  year = {2003},
  edition = {3},
  pagetotal = {632},
  langid = {russian},
}
```

### `@article` — журнальные статьи

Обязательные поля: `author`, `title`, `journal`, `volume`, `year`, `pages`, `langid`.
Опциональные: `number`.

Пример с защитой регистра имени собственного в title:
```bibtex
@article{cai1999,
  author = {Cai, X.-C. and Sarkis, M.},
  title = {A Restricted Additive {S}chwarz Preconditioner for General Sparse
           Linear Systems},
  journal = {SIAM Journal on Scientific Computing},
  volume = {21},
  number = {2},
  year = {1999},
  pages = {792--797},
  langid = {english},
}
```

### `@online` — электронные ресурсы

**Правило:** любой внешний URL, на который опирается работа, оформляется именно как `@online` запись и цитируется через `\cite{...}`. GitHub-репозитории (включая свой), документация продуктов (MATLAB, NumPy, PyTorch и т. п.), статьи в интернете, посты на Хабре, методички в сети, видеолекции, материалы конференций в открытом доступе, ответы на Stack Overflow, статьи Википедии — всё через библиографию. **Прямой `\url{https://...}` в теле работы или подстраничная сноска недопустимы.** Единственный URL вне библиографии — внутри поля `url` самой записи `@online`.

Обязательные поля: `author`, `title`, `year`, `url`, `urldate`, `langid`.

Документация продукта (коллективный автор):
```bibtex
@online{matlab2023,
  author = {{The MathWorks, Inc.}},
  title = {MATLAB Documentation},
  year = {2023},
  url = {https://www.mathworks.com/help/matlab/},
  urldate = {2026-05-21},
  langid = {english},
}
```

Личный репозиторий:
```bibtex
@online{schwartz_github,
  author = {Иванов, И. И.},
  title = {Schwartz: реализация аддитивного и мультипликативного методов Шварца
           на MATLAB},
  year = {2025},
  url = {https://github.com/username/schwartz},
  urldate = {2026-05-22},
  langid = {russian},
}
```

## 3. Поле `author`

- **Одиночный автор, кириллица**: `{Самарский, А. А.}` — фамилия, запятая, пробел, инициалы через точку-пробел.
- **Одиночный автор, латиница**: `{Toselli, A.}` — та же логика.
- **Несколько авторов**: разделяются ` and ` (с пробелами):
  `{Тихонов, А. Н. and Самарский, А. А.}`,
  `{Cai, X.-C. and Sarkis, M.}`.
- **Зарубежные авторы в русском переводе** оформляются по-русски: `{Ортега, Дж.}`, `{Голуб, Дж. and Ван Лоун, Ч.}`. Инициалы — русские сокращения (Дж. вместо J., Ч. вместо C.).
- **Корпоративный/коллективный автор** — двойные фигурные скобки, чтобы biblatex не пытался разобрать на имя и фамилию: `{{The MathWorks, Inc.}}`, `{{ГОСТ Р 7.0.100-2018}}`.
- **Себя** — так же, как любого другого: `{Иванов, И. И.}`.
- Дефисные имена сохраняют дефис: `Cai, X.-C.`, `Жидков, Н.-П.` (если так в оригинале).

## 4. Поле `title`

- Без курсива, без кавычек на верхнем уровне.
- **Защита регистра отдельных букв**: `{S}chwarz`, `{\"U}ber`, `Z{\"u}rich`. biblatex для англоязычных журнальных titles умеет переводить заглавия в lowercase — фигурные скобки фиксируют исходный регистр.
- Конечной точки не ставится — biblatex добавит сам.
- Длинные titles разрешено переносить внутри `{...}` на несколько строк с отступом (см. `cai1999`, `schwartz_github`).
- Умляуты и спецсимволы — через TeX-команды: `{\"U}`, `{\"u}`, `{\ss}`, не Unicode.
- Двоеточие в названии (подзаголовок) — оставляется как есть: `Domain Decomposition Methods: Algorithms and Theory`.

## 5. Издательство и место (`publisher`, `address`)

- `address = {Москва}` — полное название города, не «М.». Аналогично: `{Санкт-Петербург}`, не «СПб.»; `{Нижний Новгород}`, не «Н. Новгород».
- `publisher = {Наука}` — короткое название без типа издательства, если возможно.
- Если издательство не сокращается на обложке — пишется полностью: `{Изд-во МГУ}`, `{БИНОМ. Лаборатория знаний}`, `{Высшая школа}`.
- Для зарубежных изданий — оригинальное название: `{Springer}`, `{Oxford University Press}`, `{Cambridge University Press}`.

## 6. Страницы

- **Книга**: `pagetotal = {616}` — общее число страниц, без буквы «с.».
- **Статья**: `pages = {272--286}` — диапазон через двойной дефис (em-dash в исходнике). Не `272-286`, не `272—286`.
- Многотомники: тома указываются через `volume`, не в `title`.

## 7. `edition`

Просто номер: `edition = {3}`, `edition = {6}`. Стиль `gost-numeric` сам подставит «3-е изд.» / «6-е изд.» для русских записей и «3rd ed.» для английских.

## 8. `langid` — обязательно для каждой записи

Значения по факту языка источника:

| `langid` | Когда ставить |
|---|---|
| `russian` | Русскоязычные источники, включая переводы зарубежных авторов на русский |
| `english` | Англоязычные источники |
| `german` | Немецкоязычные |
| `french` | Франкоязычные |

`langid` управляет локализацией маркеров библиографии: «С.» против «pp.» против «S.», «и др.» против «et al.», «изд.» против «ed.». **Пропускать категорически нельзя** — иначе ГОСТ-стиль смешает русские и английские пометы в одной записи.

Важно: язык определяется по языку публикации, а не по гражданству автора. Книга Ортеги в переводе на русский в издательстве «Мир» — `langid = {russian}`, хотя автор американец.

## 9. Электронные ресурсы (`@online`) — детали

- `url = {https://...}` без обёртки `\url{}` — biblatex сам обработает.
- `urldate = {YYYY-MM-DD}` — дата обращения, обязательна для всех `@online`. Формат — ISO 8601.
- Для документации продуктов автор — корпоративный (`{{The MathWorks, Inc.}}`, `{{The Wolfram Research, Inc.}}`).
- Для GitHub-репозиториев — обычно сам автор кода: `{Иванов, И. И.}`.
- Если у ресурса нет явной даты публикации — `year` ставится равным году обращения, по факту просмотра.

## 10. Чего не делать

- Не используй `@misc` без крайней необходимости — он почти всегда заменяется на `@book`, `@article` или `@online`.
- Не вставляй URL в `note`/`howpublished`/`title` — для этого есть `url`.
- Не повторяй информацию из одного поля в другом (например, не пиши «изд. 3-е» в title, если есть `edition = {3}`).
- Не оставляй пустые поля — удаляй их совсем, если данных нет.
- Не используй `{и др.}` в `author` — biblatex сам сократит при необходимости. Перечисляй всех авторов до 4–5; для большего числа biblatex применит `et al.`/«и др.».

## 11. Проверки перед сдачей

- У каждой записи стоит `langid` — соответствующий языку источника.
- У всех `@online` есть `urldate` в формате `YYYY-MM-DD`.
- У всех `@book` есть `pagetotal`, у всех `@article` — `pages` с `--`.
- Все коллективные/корпоративные авторы обёрнуты в `{{...}}`.
- Все ключи `\cite{...}` из текста присутствуют в `.bib`.
- В `.bib` нет «лежащих про запас» нецитированных записей — стиль numeric-comp с `sorting=none` всё равно их не выведет, но порядок такой считается чище.
- В преамбуле работы перед `\printbibliography` ничего лишнего не добавлено; заголовок управляется `\addtoc + \section*` (как в `report.tex` эталона).
- Имена собственные в title англоязычных статей защищены `{...}`: `{S}chwarz`, `{B}anach`, `{V}olterra`.
