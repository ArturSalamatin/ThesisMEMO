## 1. Базовый принцип: панели поштучно, сшивает LaTeX

Каждая панель — **отдельный файл графики** (`panel_a.pdf`, `panel_b.pdf`, …). График в matplotlib (или другом инструменте) рисуется «чистым»: оси, кривые, подписи осей, легенда — да; литера «а», «б» **внутри картинки — нет**. Литеру под панелью ставит LaTeX через `\caption{а}` внутри `subfigure`.

Почему так:
- Панель переиспользуется. Тот же `panel_a.pdf` можно поставить и в одиночный `figure` (без литеры), и в многопанельный (с литерой), и переставить местами с соседями — буква под ней пересчитается автоматически.
- Нет рассинхронизации. Если внутри картинки зашита «а», а в подписи к рисунку панели переставили — на картинке всё ещё «а», в подписи уже «б». LaTeX-сборка такую ошибку исключает: литера живёт только в `\caption{...}` внутри `subfigure`.
- Шрифт литеры совпадает с шрифтом подписи к рисунку, а не с шрифтом matplotlib. Это важно для единого вида работы.

**Запрет**: никаких `ax.text(0.5, -0.25, 'а', transform=ax.transAxes, ...)` для литер. Подписи осей, заголовки внутри панели, аннотации над кривыми — можно, литеру а/б/в/г — нельзя.

## 2. Настройка `subcaption` в преамбуле

В `references/preamble.tex` подключено:

```latex
\usepackage{subcaption}
\captionsetup[sub]{labelformat=empty, font=normalsize, skip=2pt}
```

Расшифровка:
- `labelformat=empty` — убирает автонумерацию подподписи («(a)», «1.1»). Под панелью остаётся ровно то, что ты написал в `\caption{...}` — литера.
- `font=normalsize` — литера тем же кеглем, что основная подпись.
- `skip=2pt` — расстояние от нижней границы картинки до литеры. Если у графика широкая нижняя ось с подписью, увеличь до `4pt`–`6pt` локально через `\captionsetup[sub]{skip=4pt}` перед `\begin{figure}`.

Дополнительно ничего настраивать не нужно: одиночные `\caption{...}` для рисунка-целиком наследуют настройки из глобального `\captionsetup[figure]{...}` в преамбуле.

## 3. Идиомы по числу панелей

### 3.1. Две панели

Две `subfigure` шириной `0.48\textwidth`, между ними `\hfill`. Подходит для большинства случаев: график A — график B, до/после, эталон/результат.

```latex
\begin{figure}[h]
    \centering
    \begin{subfigure}[b]{0.48\textwidth}
        \centering
        \includegraphics[width=\linewidth]{panel_a.pdf}
        \caption{а}
    \end{subfigure}
    \hfill
    \begin{subfigure}[b]{0.48\textwidth}
        \centering
        \includegraphics[width=\linewidth]{panel_b.pdf}
        \caption{б}
    \end{subfigure}
    \caption{Тригонометрические функции:
        а~--- $\sin x$; б~--- $\cos x$}
    \label{fig:trig}
\end{figure}
```

Если панели узкие по содержанию (один график без широкой легенды), можно ужать до `0.42\textwidth` и заменить `\hfill` на `\hspace{0.04\textwidth}` для центрирования пары.

### 3.2. Три панели

Три `subfigure` по `0.31\textwidth`, между ними `\hfill`. Это предельный случай для ряда — больше трёх в одну строку при формате `extarticle 14pt` уже нечитабельно: текст внутри картинок становится мелким.

```latex
\begin{figure}[h]
    \centering
    \begin{subfigure}[b]{0.31\textwidth}
        \centering
        \includegraphics[width=\linewidth]{panel_a.pdf}
        \caption{а}
    \end{subfigure}
    \hfill
    \begin{subfigure}[b]{0.31\textwidth}
        \centering
        \includegraphics[width=\linewidth]{panel_b.pdf}
        \caption{б}
    \end{subfigure}
    \hfill
    \begin{subfigure}[b]{0.31\textwidth}
        \centering
        \includegraphics[width=\linewidth]{panel_c.pdf}
        \caption{в}
    \end{subfigure}
    \caption{Три варианта аппроксимации:
        а~--- кусочно-линейная; б~--- сплайн; в~--- эталон}
    \label{fig:approx}
\end{figure}
```

### 3.3. Четыре панели (сетка $2 \times 2$)

Базовый формат сравнения четырёх вариантов или просмотра четырёх показателей рядом. Два ряда по две `subfigure` шириной `0.48\textwidth`. Между рядами **пустая строка** (это важно — без неё `subfigure` уйдут в одну строку и переполнят `\hbox`) и `\vspace{0.6em}`, чтобы ряды не слипались по вертикали.

```latex
\begin{figure}[h]
    \centering
    \begin{subfigure}[b]{0.48\textwidth}
        \centering
        \includegraphics[width=\linewidth]{panel_a.pdf}
        \caption{а}
    \end{subfigure}
    \hfill
    \begin{subfigure}[b]{0.48\textwidth}
        \centering
        \includegraphics[width=\linewidth]{panel_b.pdf}
        \caption{б}
    \end{subfigure}

    \vspace{0.6em}

    \begin{subfigure}[b]{0.48\textwidth}
        \centering
        \includegraphics[width=\linewidth]{panel_c.pdf}
        \caption{в}
    \end{subfigure}
    \hfill
    \begin{subfigure}[b]{0.48\textwidth}
        \centering
        \includegraphics[width=\linewidth]{panel_d.pdf}
        \caption{г}
    \end{subfigure}
    \caption{Семейство гармоник:
        а~--- $\sin x$; б~--- $\cos x$;
        в~--- $\sin 2x$; г~--- $\sin x \cdot \cos x$}
    \label{fig:harm}
\end{figure}
```

### 3.4. Пять и более панелей

Сетка $2 \times 3$ (шесть панелей) или $3 \times 2$ — той же логикой: блоки рядов разделены пустой строкой и `\vspace`. Для шести панелей в ряд — ширина `0.31\textwidth`, для трёх рядов по две — `0.48\textwidth`.

Для **нечётного** числа панелей (пять, семь) последний ряд оставляй неполным, но **центрируй** через `\hspace*{\fill}` с обеих сторон:

```latex
\hspace*{\fill}
\begin{subfigure}[b]{0.31\textwidth}...\end{subfigure}
\hspace*{\fill}
```

Жёстких сеток сверх $2 \times 3$ старайся избегать — это сигнал, что рисунок надо переосмыслить (разбить на два отдельных, убрать дублирующие панели, перенести часть в приложение).

## 4. Подпись к многопанельному рисунку

Конструкция: один общий зачин (что вообще на рисунке) + двоеточие + перечисление панелей через точку с запятой. В конце точки нет (общее правило подписей).

```latex
\caption{Сравнение пяти методов выбора числа кластеров на~шумных блобах:
    а~--- кривая инерции $J(k)$; б~--- средний силуэт~$\bar s(k)$;
    в~--- индекс Дэвиса\,--\,Боулдина; г~--- индекс Калински\,--\,Харабаса;
    д~--- статистика разрыва~$\mathrm{Gap}(k)$ с~интервалами~$\pm s_k$}
```

Что важно:
- **Литера без скобок и без точки**: `а~--- ...`, а не `а)~--- ...` и не `а.~--- ...`.
- **Тильда** между литерой и `---` (неразрывный пробел).
- **Двойной дефис** `\,--\,` (а не `~---`) применяется только внутри собственных имён («Дэвис\,--\,Боулдин», «Калински\,--\,Харабас») — это уже не литеры панелей, а тире в имени.
- Перечисление панелей **разделяется `;`**, не запятой. Запятая внутри пояснения панели читается тогда без двусмысленности.
- Последний пункт **без финального символа** (ни точки, ни `;`).
- Литеры идут **в алфавитном порядке кириллицы**: а, б, в, г, д, е, ж, з, и, к (не a, b, c).

## 5. Ссылка на панель из текста

Формат: `рис.~\ref{fig:cmp},~а` — запятая после номера рисунка, тильда, литера. Без скобок вокруг литеры. В начале предложения — с прописной: `Рисунок~\ref{fig:cmp},~а`.

Несколько панелей подряд:
- две — `рис.~\ref{fig:cmp},~а и~б`;
- через перечисление — `рис.~\ref{fig:cmp},~а, б, в`.

Если ссылка на весь рисунок целиком — `рис.~\ref{fig:cmp}` без литеры. Литера обязательна только когда обсуждается конкретная панель.

## 6. Префикс меток и общие конвенции

- `\label{fig:...}` ставится **после** общего `\caption` многопанельного рисунка, **не** внутри `subfigure`. Метки отдельных панелей не нужны — ссылка идёт через `,~а` (см. раздел 5).
- Если **очень** нужно сослаться на панель из другого рисунка через `\ref` (редкий случай — обычно достаточно текстового упоминания), внутри `subfigure` после `\caption{а}` поставь `\label{fig:cmp:a}`. Префикс — `fig:общая-метка:литера`.
- TikZ-панели сшиваются той же `subfigure`-схемой: внутри `subfigure` вместо `\includegraphics` — `\begin{tikzpicture}...\end{tikzpicture}`.

## 7. Типичные ошибки

| Ошибка | Как должно быть |
|---|---|
| Литера «а» нарисована внутри картинки через `ax.text` | Картинка чистая, литера через `\caption{а}` в `subfigure` |
| `а)~---` со скобкой в подписи | `а~---` без скобки |
| Литеры латиницей: `a~---, b~---, c~---` | Кириллицей: `а~---, б~---, в~---` |
| Между рядами `subfigure` нет пустой строки | Пустая строка обязательна, иначе перелив `\hbox` |
| Финальная точка в `\caption` | Без точки |
| Перечисление панелей через запятую | Через точку с запятой |
| `\label` внутри `subfigure` без необходимости | `\label` после общего `\caption` рисунка |
| В матплотлиб литера «а» прописана крупно и жирно | Литеру ставит LaTeX обычным начертанием, кеглем подписи |

## 8. Чеклист перед сдачей многопанельного рисунка

1. Каждая панель — отдельный файл в `figures/`, литер внутри нет.
2. В преамбуле есть `subcaption` с `labelformat=empty`.
3. Литеры идут в алфавитном порядке кириллицы, без скобок, без точки.
4. Общая `\caption` имеет зачин + двоеточие + перечисление через `;`, без финальной точки.
5. `\label{fig:...}` стоит после общего `\caption`, не внутри `subfigure`.
6. Все упоминания рисунка в тексте проверены: `рис.~\ref{...}` или `рис.~\ref{...},~а`.
7. Сетка читается: при $2 \times 2$ панели не слишком узкие, оси не наезжают; между рядами есть `\vspace{0.6em}`.
8. Если панелей больше четырёх — стоит спросить, не разбить ли рисунок на два.
