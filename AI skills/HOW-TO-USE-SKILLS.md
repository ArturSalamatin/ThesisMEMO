# Как использовать AI-скиллы для Claude Code

Скилл (skill) — это набор инструкций, который превращает Claude Code в специалиста по конкретной задаче. Вместо того чтобы каждый раз объяснять, как оформлять документ или писать код, вы подключаете готовый скилл, и Claude следует всем правилам автоматически.

В этой папке лежит скилл **gost-tex-report** — он умеет писать и оформлять научно-технические документы на LaTeX (курсовые, ВКР, отчёты по НИР) по ГОСТ.

---

## Что внутри скилла

```
gost-tex-report/
├── SKILL.md                        главный файл с правилами
└── references/
    ├── preamble.tex                готовая LaTeX-преамбула
    ├── ai-tells.md                 стоп-лист фраз, выдающих ИИ-генерацию
    ├── author-style.md             протокол подстройки под ваш стиль письма
    ├── bibliography.md             правила библиографии по ГОСТ
    ├── figures.md                  правила оформления рисунков
    └── examples/
        ├── report.tex              эталонный пример — обвес работы
        ├── content.tex             эталонный пример — тело работы
        └── references.bib          эталонный пример — библиография
```

Скилл копируется целиком — папка `gost-tex-report/` со всем содержимым. Файлы из `references/` нужны Claude для работы, не удаляйте их.

---

## Как подключить скилл к своему проекту

### Шаг 1. Убедитесь, что у вас установлен Claude Code

Claude Code — это CLI-инструмент (или расширение для VS Code / JetBrains), который запускает Claude прямо в вашем проекте. Если вы читаете эту инструкцию внутри Claude Code — он у вас уже есть.

### Шаг 2. Скопируйте папку скилла в свой проект

В корне вашего LaTeX-проекта создайте папку `.claude/skills/` и скопируйте туда папку `gost-tex-report/` целиком:

```
ваш-проект/
├── .claude/
│   └── skills/
│       └── gost-tex-report/
│           ├── SKILL.md
│           └── references/
│               └── ...
├── report.tex
├── content.tex
├── references.bib
└── figures/
```

На Windows это можно сделать командой:

```powershell
# Из корня вашего LaTeX-проекта:
New-Item -ItemType Directory -Force ".claude\skills"
Copy-Item -Recurse "ПУТЬ\К\gost-tex-report" ".claude\skills\gost-tex-report"
```

На Linux/macOS:

```bash
# Из корня вашего LaTeX-проекта:
mkdir -p .claude/skills
cp -r ПУТЬ/К/gost-tex-report .claude/skills/
```

### Шаг 3. Начните работу

Откройте проект в Claude Code (или в VS Code с расширением Claude Code) и просто попросите написать или оформить текст. Claude автоматически увидит скилл и будет следовать его правилам.

---

## Альтернативный способ — без копирования

Если не хотите копировать файлы, можно в начале разговора с Claude сказать:

> Прочитай файл `ПУТЬ/К/gost-tex-report/SKILL.md` и следуй этим правилам.

Claude прочитает инструкции и будет им следовать в рамках текущей сессии. Минус: это нужно делать в каждой новой сессии.

---

## Глобальная установка (для всех проектов)

Если вы часто пишете LaTeX-документы и хотите, чтобы скилл был доступен во всех проектах:

```powershell
# Windows
Copy-Item -Recurse "ПУТЬ\К\gost-tex-report" "$env:USERPROFILE\.claude\skills\gost-tex-report"
```

```bash
# Linux / macOS
cp -r ПУТЬ/К/gost-tex-report ~/.claude/skills/
```

## Что можно попросить Claude, когда скилл активен

Примеры запросов:

- «Напиши введение для курсовой на тему [...]»
- «Оформи этот раздел по ГОСТ»
- «Создай преамбулу для нового документа»
- «Добавь библиографическую запись для книги [...]»
- «Допиши заключение»
- «Сделай рисунок из двух панелей (а, б) с подписями»
- «Проверь текст на LLM-маркеры»
- «Вот мои заметки и черновик — напиши по ним раздел "Результаты"»

### Подстройка под ваш стиль

Скилл умеет подстраиваться под ваш личный стиль письма. Для этого приложите к разговору свои материалы: черновики, конспекты, предыдущие работы, заметки. Claude проанализирует вашу манеру изложения и будет писать новый текст в том же стиле.

### Борьба с «ИИ-шным» текстом

Главная фишка скилла — текст не должен выглядеть как сгенерированный. Claude будет избегать канцеляризмов, шаблонных связок, тройных перечислений и прочих маркеров машинной генерации. Текст будет проходить проверки на антиплагиат и ИИ-контент.

## Частые вопросы

**Нужен ли LaTeX на компьютере?**
Claude пишет `.tex`-файлы. Чтобы скомпилировать их в PDF, нужен LaTeX (например, MiKTeX или TeX Live). Но сам скилл работает без компилятора — он только создаёт и редактирует текстовые файлы.

**Можно ли использовать скилл для статей, а не курсовых?**
Да. Скилл ориентирован на ГОСТ 7.32 (отчёты, курсовые, ВКР), но правила типографики, библиографии и борьбы с ИИ-маркерами универсальны.

**Что если Claude не видит скилл?**
Убедитесь, что папка `gost-tex-report/` лежит в `.claude/skills/` внутри проекта или в `~/.claude/skills/` глобально. Файл должен называться именно `SKILL.md` (заглавными буквами).

**Можно ли менять правила скилла?**
Да. `SKILL.md` — обычный текстовый файл. Вы можете отредактировать его под свои нужды: убрать ненужные разделы, добавить свои правила, изменить шаблоны.
